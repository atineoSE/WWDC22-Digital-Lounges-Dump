*Thread Reply:* It‚Äôs important to prevent ‚Äúreplay attacks‚Äù when your server is receiving tokens. iOS and macOS won‚Äôt ever send a token multiple times, but your server still needs to prevent this behavior.

Tommy P (Apple)
2022-06-09 18:43:03

*Thread Reply:* Neither Apple nor your token issuer gets to see what the client is accessing! There are two cool ways this is made private:

  1. The token challenge includes the server name, but before the token request is sent to the Apple attester or the issuer, the RSA blinding operation obfuscates all of the content.   2. Tokens are fetched in batches and are cached by the client, so not every token issuance corresponds in time to a server requesting a token.
